# Simulated signatures for label-pairing examples

A named list of three simulated \`OmicSignature\` objects designed to
demonstrate explicit \`label_pairing\` control in
\`compare_omic_signatures()\`. Each object contains two label-specific
signatures of 10 features each and a 100-row \`difexp\` table.

## Usage

``` r
data(compare_label_pairing_example)
```

## Format

A named list with three \`OmicSignature\` objects:

- signature_a:

  Simulated signature with \`treated\` and \`control\` labels.

- signature_b:

  Simulated signature with \`up\` and \`down\` labels.

- signature_c:

  Simulated signature with \`resistant\` and \`sensitive\` labels.

## Source

Generated by \`data-raw/create_compare_label_pairing_data.R\`.

## Details

The simulated overlap structure makes \`signature_a\`'s \`treated\`
label most similar to \`signature_b\`'s \`up\` label, then to
\`signature_c\`'s \`resistant\` label. Likewise, \`signature_a\`'s
\`control\` label is most similar to \`signature_b\`'s \`down\` label,
then to \`signature_c\`'s \`sensitive\` label.
